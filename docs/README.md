# Docs

Project documentation for human readers.

Use this directory for operator guides, checklists, and release notes.
Keep docs concise, current, and executable as written.

## Current docs

- `overview.md`: MVP scope, done criteria, and explicit non-goals
- `architecture.md`: runtime boundaries, invariants, and system contracts
- `content-model.md`: content types, publication rules, and editing workflow
- `frontend.md`: template/HTMX/CSS/accessibility contracts
- `operations.md`: local run/test/build/deploy/log/rollback commands
- `deployment.md`: end-to-end VPS provisioning, DNS, first deploy, smoke validation, rollback
- `deploy_checklist.md`: production deploy checklist (pre-checks, deploy, verification, rollback)
